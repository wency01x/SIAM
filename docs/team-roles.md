# Team Roles - Sprint 1

Project: Lattice v1.0
Sprint: Sprint 1

## Role Assignments

| Team Member | Primary Role | Responsibilities |
|---|---|---|
| Contributor 1 | Technical Lead and Backend Engineer | Own authentication/session architecture (US-01), base currency service integration (US-03), code reviews for backend modules |
| Contributor 2 | Security Engineer | Own RBAC enforcement design and implementation (US-02), security validation for protected routes and restricted settings |
| Contributor 3 | Product and Documentation Lead | Own settings page UX/spec alignment (US-12), maintain sprint docs and acceptance criteria traceability |
| Contributor 4 | QA and Observability Engineer | Own audit log implementation tests (US-13), smoke test coverage, regression checks in CI |

## Working Agreements
- Contributor 1 approves backend architecture changes affecting auth, roles, or data access.
- Contributor 2 signs off on security controls before story closure for US-01 and US-02.
- Contributor 3 keeps backlog and sprint documents synchronized with completed work.
- Contributor 4 blocks story completion if critical test coverage is missing.

## Collaboration Cadence
- Daily standup: progress, blockers, risk updates.
- Mid-sprint review: validate Sprint Goal and scope stability.
- End-of-sprint demo: show working auth, RBAC, settings, currency switch, and audit logs.
