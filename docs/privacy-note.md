# Privacy Note

This Privacy Note explains how data is handled within the application, focusing on transparency regarding data collection, usage, retention, and user rights.

## 1. Data Collected
- **Account Information**: Usernames, email addresses, and encrypted passwords for authentication.
- **Usage Data**: Standard access logs, timestamps, and interaction metrics (even in demonstration or testing environments).
- **Session Data**: Temporary tokens or session identifiers to maintain user state.

## 2. Purpose of Collection (Why)
- **Service Provisioning**: To authenticate users, authorize actions, and deliver the core functionality of the application.
- **Security & Integrity**: To detect and prevent fraudulent activities, unauthorized access, and security breaches.
- **Improvement & Analytics**: To monitor application performance, troubleshoot issues, and enhance user experience.

## 3. Data Retention
- **Active Accounts**: Data is retained as long as the user's account is active to provide ongoing services.
- **Logs and Analytics**: System logs are retained for a limited period (e.g., 30 days) for security auditing and debugging purposes before being purged.
- **Demonstration Environments**: In demo or test deployments, data may be routinely cleared or reset to maintain a clean state.

## 4. User Rights
Even in a demonstration context, users have the following rights regarding their data:
- **Right to Access**: Users can request to see the data collected about them.
- **Right to Rectification**: Users can request corrections to inaccurate or incomplete data.
- **Right to Erasure (Right to be Forgotten)**: Users can request the deletion of their data from the system.
- **Right to Restrict Processing**: Users can request limitations on how their data is used.

*For any privacy-related inquiries or to exercise these rights, please contact the project maintainers.*
